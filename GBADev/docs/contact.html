Contacting the Author

Contacting the author is easy. Let me know what you think of HAM, file bug reports etc, but please, follow these rules:

  • if all you want to tell me is that HAM sucks, please refrain from sending me an email and stop using it instead. That might help you get over it a lot better
  • if you have a question about how X works in HAM, please subscribe to the HAMDev mailing list first and check the list archives on http://groups.yahoo.com/group/hamdev first. It is likely that you will already find your answer there.
  • also make sure you tried to get help in the HAM FAQ section first.
